# Reporting Security Issues in AIISP

If you believe you have found a security issue in an AIISP specification, schema, reference implementation, or settlement contract design, please report it privately to the editor at `x@97115104.com` rather than filing a public issue.

Please include:

- A description of the issue.
- Reproduction steps where applicable.
- Your assessment of the impact.
- Any suggested mitigation.
- The affected spec, schema, example, or reference component.

The editor commits to acknowledging the report within 72 hours and to publishing a coordinated disclosure once a fix or mitigation is available. Reporters will be credited in the relevant spec revision history unless they request otherwise.

For non-security errata such as typos, broken links, arithmetic errors, or factual mistakes, please use the public Errata issue template instead.
