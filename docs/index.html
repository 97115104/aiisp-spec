1. What this is

AIISP-2 is the active Human Data Collective v2 draft. It specifies a contributor-aligned inference routing and settlement layer for category-specific models, contributor-operated inference nodes, accepted data contributions, accepted DPO feedback pairs, event-based HDCT issuance, environmental accounting, and public settlement records.

AIISP-1 remains published as a historical and provider-facing draft for centralized request-cost disclosure. AIISP-2 changes the counterparty model: the router, nodes, contributors, verifiers, and settlement contract become the core protocol surface.

2. The event rule

HDCT issuance is event-based only. A recorded event cannot be reminted.

  • accepted data contribution - a verifier accepts a human data contribution.
  • verified inference served - a node serves inference, or a batch settles a defined inference unit.
  • accepted DPO pair - a verifier accepts a preference pair.

AIISP-2 does not define recurring royalties for old data contributions. Later model use does not remint a prior contribution event.
